⬡
Xephyr

インストール

bash
# Podman + Xephyr
sudo apt install -y podman xserver-xephyr

# NVIDIA Container Toolkit
curl -fsSL https://nvidia.github.io/libnvidia-container/gpgkey \
  | sudo gpg --dearmor -o /usr/share/keyrings/nvidia-container-toolkit-keyring.gpg

curl -s -L https://nvidia.github.io/libnvidia-container/stable/deb/nvidia-container-toolkit.list \
  | sed 's#deb https://#deb [signed-by=/usr/share/keyrings/nvidia-container-toolkit-keyring.gpg] https://#g' \
  | sudo tee /etc/apt/sources.list.d/nvidia-container-toolkit.list

sudo apt update && sudo apt install -y nvidia-container-toolkit

# CDI デバイス定義生成(rootless Podman から GPU を認識させる)
sudo nvidia-ctk cdi generate --output=/etc/cdi/nvidia.yaml
ℹ
CDI (Container Device Interface) を使うことで --device nvidia.com/gpu=all の記法が使える。rootless Podman でも安全に GPU をパススルー可能。
02

Dockerfile

📦
Windsurf / Antigravity は 公式 .deb を curl + dpkg -i で導入。外部リポジトリ登録なし。Ollama はバイナリのみ収録し、モデルデータは Volume で管理。
dockerfile
FROM debian:12-slim

# ── base deps ──────────────────────────────────
RUN apt-get update && apt-get install -y --no-install-recommends \
  # X / WM / D-Bus
  openbox dbus-x11 xdg-utils xauth \
  # Electron / Chromium 共通ライブラリ
  libgbm1 libgl1-mesa-dri \
  libnss3 libatk1.0-0 libatk-bridge2.0-0 \
  libgdk-pixbuf2.0-0 libgtk-3-0 \
  libasound2 libxss1 libxtst6 \
  # フォント
  fonts-noto-cjk fonts-noto-color-emoji \
  # ユーティリティ
  ca-certificates curl wget gnupg \
  # ビルドツール
  build-essential git \
  # Python
  python3 python3-pip python3-venv \
  && rm -rf /var/lib/apt/lists/*

# ── Node.js (nvm) ──────────────────────────────
ENV NVM_DIR=/opt/nvm NODE_VERSION=22
RUN mkdir -p $NVM_DIR \
  && curl -fsSL https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.1/install.sh | bash \
  && . $NVM_DIR/nvm.sh \
  && nvm install $NODE_VERSION && nvm alias default $NODE_VERSION
ENV PATH="$NVM_DIR/versions/node/v${NODE_VERSION}/bin:$PATH"

# ── Rust ───────────────────────────────────────
ENV RUSTUP_HOME=/opt/rustup CARGO_HOME=/opt/cargo
RUN curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs \
  | sh -s -- -y --no-modify-path
ENV PATH="$CARGO_HOME/bin:$PATH"

# ── CLI AI agents ──────────────────────────────
RUN npm install -g \
  @anthropic-ai/claude-code \
  @openai/codex \
  @google/gemini-cli \
  opencode-ai

RUN pip3 install --break-system-packages aider-chat

# ── Ollama (binary only, models → Volume) ──────
RUN curl -fsSL https://ollama.com/install.sh | sh

# ── Windsurf (.deb) ────────────────────────────
ARG WINDSURF_DEB=https://windsurf-stable.codeiumdata.com/linux-x86_64/windsurf_latest_amd64.deb
RUN curl -fsSL $WINDSURF_DEB -o /tmp/windsurf.deb \
  && dpkg -i /tmp/windsurf.deb || apt-get install -fy \
  && rm /tmp/windsurf.deb

# ── Antigravity (.deb) — URL 公開後に差し替え ──
ARG ANTIGRAVITY_DEB=https://antigravity.google/download/antigravity_latest_amd64.deb
RUN curl -fsSL $ANTIGRAVITY_DEB -o /tmp/antigravity.deb \
  && dpkg -i /tmp/antigravity.deb || apt-get install -fy \
  && rm /tmp/antigravity.deb

# ── CUDA Toolkit (runtime のみ) ─────────────────
# ホストの nvidia-smi で確認したドライバに合わせてバージョンを調整
RUN apt-get update && apt-get install -y --no-install-recommends \
  cuda-cudart-12-4 libcublas-12-4 \
  && rm -rf /var/lib/apt/lists/*

# ── user ───────────────────────────────────────
RUN useradd -m -s /bin/bash agent
USER agent
WORKDIR /home/agent/work

COPY --chown=agent:agent entrypoint.sh /home/agent/entrypoint.sh
RUN chmod +x /home/agent/entrypoint.sh

ENTRYPOINT ["/home/agent/entrypoint.sh"]
CMD ["bash"]
03

entrypoint.sh

bash
#!/bin/bash
set -e

# D-Bus セッション起動 (Electron 必須)
eval $(dbus-launch --sh-syntax)
export DBUS_SESSION_BUS_ADDRESS

# WM 起動
openbox &

# GPU 確認
[ -e /dev/dri/renderD128 ] && export LIBGL_ALWAYS_SOFTWARE=0 \
                            || export LIBGL_ALWAYS_SOFTWARE=1

sleep 0.5

exec "${@:-bash}"

05

Build & Run

イメージビルド

bash
podman build -t agent-sandbox:latest .

Xephyr 起動 — ホスト側・別ターミナル

bash
# XWayland 上でネスト X を独立ウィンドウとして起動
# -ac: コンテナからの接続を許可
Xephyr :10 -screen 1920x1080 -ac &

コンテナ起動

bash
podman run -it --rm \
  # Xephyr のみ公開 (ホスト :0 は渡さない)
  -e DISPLAY=:10 \
  -v /tmp/.X11-unix/X10:/tmp/.X11-unix/X10:ro \
  \
  # Chromium / Electron が必要とする共有メモリ
  -v /dev/shm:/dev/shm \
  \
  # NVIDIA GPU (CDI)
  --device nvidia.com/gpu=all \
  \
  # ネットワーク隔離 (NAT)
  --network slirp4netns \
  \
  # 永続 Volume
  -v agent-workspace:/home/agent/work \
  -v ollama-models:/root/.ollama/models \
  \
  agent-sandbox:latest \
  windsurf    # または antigravity / bash
✓
ホスト DISPLAY(:0) は渡していない。 コンテナは Xephyr(:10) のみアクセス可能。ホストのキー入力・画面・クリップボードには到達できない。
06

Ollama モデル管理

⚠
モデルデータはイメージに焼かず ollama-models Volume で管理。ビルドを軽量に保ちつつ、モデルを永続利用できる。

モデルの pull (コンテナ内)

bash
# コンテナ内で ollama サーバー起動 → モデル取得
ollama serve &
sleep 2

# 軽量モデル例 (~2GB)
ollama pull qwen2.5-coder:3b

# 高性能モデル例 (~5GB, VRAM 6GB 以上推奨)
ollama pull qwen2.5-coder:7b

モデルサイズ目安

text
qwen2.5-coder:3b     ~2 GB    VRAM 3GB+   軽量・高速
qwen2.5-coder:7b     ~5 GB    VRAM 6GB+   バランス型
llama3.2:3b          ~2 GB    VRAM 3GB+   汎用
codellama:7b         ~4 GB    VRAM 6GB+   コーディング特化
⚠
VRAM はエージェント (Windsurf / Antigravity の GPU 合成) と Ollama で共有される。合計 VRAM に収まるモデルを選択すること。nvidia-smi で使用量を確認しながら運用推奨。
07

隔離まとめ

リソース コンテナからのアクセス 方法
Host Filesystem BLOCKED マウントなし
Host Display (:0) BLOCKED Xephyr :10 のみ渡す
Host Keyboard / Clipboard BLOCKED DISPLAY 分離による
Host Network NAT only slirp4netns
NVIDIA GPU PASSTHROUGH CDI / --device nvidia.com/gpu=all
agent-workspace Volume READ / WRITE 作業ファイル専用
ollama-models Volume READ / WRITE モデルデータ専用
Host Kernel SHARED namespace 分離 (Podman)
08

Tips

Xephyr 解像度変更

bash
# HiDPI (2560x1440)
Xephyr :10 -screen 2560x1440 -dpi 144 -ac &

ワークスペースのバックアップ

bash
podman run --rm \
  -v agent-workspace:/work:ro \
  -v $(pwd):/out \
  debian:12-slim \
  tar czf /out/workspace-$(date +%Y%m%d).tar.gz -C /work .

Antigravity 権限制限 (ゲスト内設定)

text
Settings > Terminal
  Auto Execution  →  Request Review   # 全コマンドを承認制に
  Deny List       →  rm -rf, curl, wget (任意)

Settings > Browser
  URL Allowlist   →  localhost:*      # 外部遮断が必要な場合

CUDA バージョン確認

bash
# ホスト側で driver バージョンを確認してから Dockerfile を調整
nvidia-smi | grep "CUDA Version"
⚠
Dockerfile 内の cuda-cudart-12-4 のバージョン番号は、ホストの NVIDIA ドライバーが対応する CUDA バージョンに合わせること。
