🏗️ アーキテクチャ (Architecture)

Linux (Ubuntu/Fedora等) の上で Podman コンテナを動かし、Xephyr という「ウィンドウの中で動く X サーバー」を介して画面を表示します。これにより、エージェントがホストのクリップボードや他のウィンドウに勝手に触るのを防ぎます。

02

🏗️ ホスト側の準備 (Host Preparation)

  • x86_64 (amd64) アーキテクチャ必須 — ARM Linux は非対応
  • NVIDIA GPU 搭載マシン（VRAM 8GB 以上を推奨）
  • Podman 4.1.0 以上（CDI サポートのため 5.0+ 推奨）
  • xserver-xephyr (または単に xephyr) パッケージ

1. ツール一式のインストール

bash
sudo apt update
sudo apt install -y podman podman-compose xserver-xephyr x11-xserver-utils nvidia-container-toolkit

2. NVIDIA CDI の生成

bash
sudo nvidia-ctk cdi generate --output=/etc/cdi/nvidia.yaml
# 確認
podman run --rm --device nvidia.com/gpu=all debian:12-slim nvidia-smi
03

📦 Dockerfile の作成

エージェントを動かすためのコンテナイメージを作成します。Node.js や Python、そして表示用の Openbox を詰め込みます。

Dockerfile
FROM debian:12-slim

RUN apt-get update && apt-get install -y --no-install-recommends \
    bash curl wget git procps python3 python3-pip python3-venv \
    xterm openbox dbus-x11 libnss3 libatk1.0-0 libatk-bridge2.0-0 \
    libcups2 libdrm2 libxkbcommon0 libxcomposite1 libxdamage1 \
    libxrandr2 libgbm1 libasound2 zstd gnupg unzip \
    locales fonts-noto-cjk fonts-noto-color-emoji \
    && sed -i 's/# ja_JP.UTF-8/ja_JP.UTF-8/' /etc/locale.gen \
    && locale-gen \
    && rm -rf /var/lib/apt/lists/*

ENV LANG=ja_JP.UTF-8 LC_ALL=ja_JP.UTF-8

# NOTE: cuda-cudart-12-4 はスキップ (NVIDIA GPU なし)
# GPU あり環境では下記を有効化:
# RUN curl -fsSL https://developer.download.nvidia.com/compute/cuda/repos/debian12/x86_64/cuda-keyring_1.1-1_all.deb \
#     -o /tmp/cuda-keyring.deb && dpkg -i /tmp/cuda-keyring.deb && rm /tmp/cuda-keyring.deb \
#     && apt-get update && apt-get install -y --no-install-recommends cuda-cudart-12-4 \
#     && rm -rf /var/lib/apt/lists/*

# ── Node.js (NVM) ───────────────────────────────
ENV NVM_DIR=/opt/nvm NODE_VERSION=22
RUN mkdir -p $NVM_DIR \
  && curl -fsSL https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.1/install.sh | bash \
  && . $NVM_DIR/nvm.sh \
  && nvm install $NODE_VERSION && nvm alias default $NODE_VERSION
ENV PATH="$NVM_DIR/versions/node/v${NODE_VERSION}/bin:$PATH"

# ── npm CLI tools ───────────────────────────────
RUN npm install -g \
  @google/gemini-cli \
  opencode-ai \
  codex-cli

# ── Windsurf (apt repo) ─────────────────────────
RUN mkdir -p /etc/apt/keyrings \
  && wget -qO- "https://windsurf-stable.codeiumdata.com/wVxQEIWkwPUEAGf3/windsurf.gpg" \
     | gpg --dearmor > /etc/apt/keyrings/windsurf-stable.gpg \
  && echo "deb [arch=amd64 signed-by=/etc/apt/keyrings/windsurf-stable.gpg] https://windsurf-stable.codeiumdata.com/wVxQEIWkwPUEAGf3/apt stable main" \
     > /etc/apt/sources.list.d/windsurf.list \
  && apt-get update \
  && apt-get install -y --no-install-recommends windsurf \
  && rm -rf /var/lib/apt/lists/*

RUN useradd -m -s /bin/bash agent
USER agent
WORKDIR /home/agent/work
ENV PATH="/home/agent/.local/bin:$PATH"

# ── Claude Code ──
RUN curl -fsSL https://claude.ai/install.sh | bash

# ── Aider ──
RUN curl -fsSL https://aider.chat/install.sh | sh

# ── Kiro ── (コマンド名は kiro-cli)
RUN curl -fsSL https://cli.kiro.dev/install | bash

COPY entrypoint.sh /usr/local/bin/
ENTRYPOINT ["/usr/local/bin/entrypoint.sh"]
04

🏗️ Entrypoint の作成

コンテナ起動時に X サーバーとの接続やウィンドウマネージャーをふわっと準備するスクリプトです。

bash (entrypoint.sh)
#!/bin/bash
export DISPLAY=:10
# D-Bus の準備
eval $(dbus-launch --sh-syntax)
# Openbox をバックグラウンドで起動
openbox &
# ターミナルを表示して待機
xterm -maximized -e bash
05

🚀 起動とログイン

準備ができたら、まずホスト側で Xephyr を起動し、その後にコンテナを立ち上げます。

💡
compose.yml を使った管理がおすすめです。設定例はリポジトリのルートにある compose.yml を見てみてくださいね🐾

1. ホスト側で Xephyr 起動

bash
# 1280x800 のウィンドウを作って X11 ディスプレイ番号 :10 を割り当て
Xephyr :10 -screen 1280x800 -ac &

2. コンテナのビルドと起動

bash
podman build -t agent-sandbox .
podman run -it --rm \
  --device nvidia.com/gpu=all \
  --network slirp4netns \
  -e DISPLAY=:10 \
  -v /tmp/.X11-unix/X10:/tmp/.X11-unix/X10:ro \
  -v agent-workspace:/home/agent/work \
  -v claude-auth:/home/agent/.config/claude \
  -v windsurf-auth:/home/agent/.config/windsurf \
  -v kiro-auth:/home/agent/.kiro \
  --env-file ~/.config/agent-sandbox/.env \
  agent-sandbox
🔑
API キーの注入: Dockerfile に API キーを書いてはいけません！ --env-file で安全に渡しましょう✨
# ~/.config/agent-sandbox/.env (chmod 600 で保護！)
ANTHROPIC_API_KEY=sk-ant-...
GEMINI_API_KEY=AIza...
OPENAI_API_KEY=sk-...

mkdir -p ~/.config/agent-sandbox
chmod 700 ~/.config/agent-sandbox

🔐 OAuth ログイン (ブラウザ認証)

API キー不要でブラウザ OAuth を使う場合は、コンテナ内でブラウザが Xephyr に表示されます。

bash — コンテナ内
# Claude Code — ブラウザが Xephyr に開いて OAuth 完了
claude login

# Kiro — ブラウザ OAuth (AWS SSO / GitHub)
kiro-cli login

# Gemini CLI — Google アカウント認証
gemini auth login
💡
認証情報の永続化: -v claude-auth:/home/agent/.config/claude 等の named volume に認証トークンが保存されます。コンテナを再起動しても再ログイン不要です✨
ボリュームを初めて作る場合:
podman volume create claude-auth
podman volume create windsurf-auth
podman volume create kiro-auth

🦙 モデルの準備 (Ollama)

コンテナ内、またはホスト側で Ollama を動かしてエージェントに脳みそを与えてあげましょう。コンテナ内に閉じ込める場合は、ボリュームを忘れずに！

bash
# コンテナ内で Ollama インストール
curl -fsSL https://ollama.com/install.sh | bash
# 推奨モデル（コーディング特化）をプル
ollama pull qwen2.5-coder:7b

モデルサイズ目安

text

llama3.2:3b          ~2 GB    VRAM 3GB+   汎用

⚠
GPUのメモリ（VRAM）は、AIエージェントとOllamaで仲良く分け合って使います GPUの限界を超えないようなサイズのモデルを選んであげてくださいね📊 nvidia-smi コマンドで時々様子を見てあげると安心です✨

隔離の動作確認コマンド

bash — コンテナ内
# ホスト FS が見えないことを確認 (agent ディレクトリのみ表示される)
ls /home

# ネットワーク NAT 経由で外に出られることを確認
curl -s -o /dev/null -w "%{http_code}" https://example.com   # → 200

# ホスト名がコンテナ固有値であることを確認
hostname   # → コンテナ ID (ホスト名と異なる)

# GPU が使えることを確認 (NVIDIA 環境)
nvidia-smi

🔍 トラブルシューティング

コンテナ内で "cannot open display :10" が出る

Xephyr が起動していないか、DISPLAY の番号がずれています。

bash
# ホスト側：Xephyr を起動
Xephyr :10 -screen 1280x800 &
# Xephyr ウィンドウ内に別ターミナルが開けば OK
DISPLAY=:10 xterm &
コンテナ内で nvidia-smi が見つからない / GPU が認識されない

CDI 定義が未生成か、Podman が 4.1.0 未満の可能性があります。

bash
# CDI ファイル確認
ls -la /etc/cdi/nvidia.yaml
# 再生成
sudo nvidia-ctk cdi generate --output=/etc/cdi/nvidia.yaml
# GPU 認識テスト
podman run --rm --device nvidia.com/gpu=all debian:12-slim nvidia-smi
Xephyr ウィンドウが真っ黒のまま何も表示されない

openbox または dbus の起動に失敗しています。

bash
# コンテナのログを確認
podman logs agent-sandbox
# openbox を手動で試す（コンテナ内）
podman exec -it agent-sandbox bash -c "DISPLAY=:10 openbox"
CUDA バージョンのミスマッチエラーが出る

Dockerfile の cuda-cudart-12-4 とホストドライバーの CUDA バージョンを合わせてください。

bash
# ホスト側のバージョンを確認
nvidia-smi | grep "CUDA Version"
# 例: 12.6 → Dockerfile 内の 12-4 を 12-6 に変更してリビルド
podman build --no-cache -t agent-sandbox:latest .
podman pull / ollama pull が途中で止まる

Ctrl-C で中断して再実行すると続きからダウンロードされます。

bash
ollama pull qwen2.5-coder:7b  # 再実行で続きから再開
