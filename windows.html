🏗️ 全体の仕組み

ℹ
WSLg ってなに？🤔: Windows 11 に最初から入っている、Linuxの画面をWindowsに出してくれる賢い機能です📺 特別な設定をしなくてもAIの画面がポンッと出るのでとっても便利ですが、Linux版のXephyrほど強力なガードではないので気をつけてくださいね💦
01

🌊 WSL2 を入れよう！

⚠
大切なお知らせ: この手順は Windows 11 が必要です🪟 Windows 10 の方は WSLg が使えないので、Option C（Xephyrを使う方法）をおすすめします✨

PowerShell (管理者) で実行

powershell — 管理者
# WSL2 + Ubuntu 24.04 をインストール
wsl --install -d Ubuntu-24.04

# インストール後は再起動が必要
Restart-Computer

再起動後 — Ubuntu 初期設定

powershell
# WSL2 がデフォルトになっていることを確認
wsl --set-default-version 2
wsl --list --verbose   # VERSION が 2 であることを確認
bash — WSL2 Ubuntu 内
# Ubuntu 初回起動でユーザー名・パスワードを設定後
sudo apt update && sudo apt upgrade -y

.wslconfig でリソース制限 (任意)

ini — %USERPROFILE%\.wslconfig
[wsl2]
memory=8GB          # WSL2 に割り当てる最大 RAM
processors=4        # 使用する CPU コア数
swap=4GB
gpuMemoryFraction=0.8   # GPU メモリ割り当て比率
02

🎮 NVIDIA/GPU の準備

✓
NVIDIAドライバーはWindows側に入れるだけでOKです✨ WSL2（Linux）の中に間違えてドライバーを入れないように気をつけてくださいね！WSL2の中には nvidia-container-toolkit だけを入れます📦

1. Windows 側 NVIDIA ドライバー確認

powershell
# Windows 側で CUDA 対応ドライバーが入っているか確認
nvidia-smi

# 未インストールの場合は winget で
winget install NVIDIA.CUDA

2. WSL2 内で nvidia-container-toolkit をインストール

bash — WSL2
# NVIDIA Container Toolkit リポジトリ追加
curl -fsSL https://nvidia.github.io/libnvidia-container/gpgkey \
  | sudo gpg --dearmor -o /usr/share/keyrings/nvidia-container-toolkit-keyring.gpg

curl -s -L https://nvidia.github.io/libnvidia-container/stable/deb/nvidia-container-toolkit.list \
  | sed 's#deb https://#deb [signed-by=/usr/share/keyrings/nvidia-container-toolkit-keyring.gpg] https://#g' \
  | sudo tee /etc/apt/sources.list.d/nvidia-container-toolkit.list

sudo apt update && sudo apt install -y nvidia-container-toolkit

# CDI デバイス定義生成
sudo nvidia-ctk cdi generate --output=/etc/cdi/nvidia.yaml

# WSL2 内から GPU が見えることを確認
nvidia-smi
03

📦 Podman を入れよう！

bash — WSL2
# Ubuntu 24.04 は podman が公式リポジトリにある
sudo apt install -y podman

# バージョン確認
podman --version

# rootless 動作確認
podman info | grep -i rootless

📝 Dockerfile

📦
Linux版のDockerfileと違うのは DISPLAY の設定だけ です🐧 Xephyrがいらない分、少しスッキリしていますね✨ cuda-cudart-12-4 の数字は、Windows側のドライバーと仲良しなバージョンに合わせてあげてください🤖
dockerfile
FROM debian:12-slim

# ── base deps ─────────────────────────────────
RUN apt-get update && apt-get install -y --no-install-recommends \
  openbox dbus-x11 xdg-utils xauth \
  libgbm1 libgl1-mesa-dri \
  libnss3 libatk1.0-0 libatk-bridge2.0-0 \
  libgdk-pixbuf2.0-0 libgtk-3-0 \
  libasound2 libxss1 libxtst6 \
  fonts-noto-cjk fonts-noto-color-emoji locales \
  ca-certificates curl wget gnupg jq yq ripgrep fd-find vim zstd \
  # ブラウザ (Antigravity 用)
  chromium \
  # デザイン / 図表作成
  graphviz inkscape \
  build-essential git \
  python3 python3-pip python3-venv \
  && sed -i 's/# ja_JP.UTF-8/ja_JP.UTF-8/' /etc/locale.gen \
  && locale-gen \
  && rm -rf /var/lib/apt/lists/*

ENV LANG=ja_JP.UTF-8 LC_ALL=ja_JP.UTF-8

# ── Node.js (nvm) ─────────────────────────────
ENV NVM_DIR=/opt/nvm NODE_VERSION=22
RUN mkdir -p $NVM_DIR \
  && curl -fsSL https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.1/install.sh | bash \
  && . $NVM_DIR/nvm.sh \
  && nvm install $NODE_VERSION && nvm alias default $NODE_VERSION
ENV PATH="$NVM_DIR/versions/node/v${NODE_VERSION}/bin:$PATH"

# ── Rust ──────────────────────────────────────
ENV RUSTUP_HOME=/opt/rustup CARGO_HOME=/opt/cargo
RUN curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs \
  | sh -s -- -y --no-modify-path
ENV PATH="$CARGO_HOME/bin:$PATH"

# ── CLI AI agents ─────────────────────────────
RUN npm install -g \
  @google/gemini-cli \
  opencode-ai \
  codex-cli \
  && curl -fsSL https://claude.ai/install.sh | bash \
  && curl -fsSL https://github.com/block/goose/releases/latest/download/install.sh | bash

RUN curl -LsSf https://aider.chat/install.sh | sh

# ── Ollama (binary only) ──────────────────────
RUN curl -fsSL https://ollama.com/install.sh | sh

# ── Windsurf (apt repo) ────────────────────────
RUN mkdir -p /etc/apt/keyrings \
  && wget -qO- "https://windsurf-stable.codeiumdata.com/wVxQEIWkwPUEAGf3/windsurf.gpg" \
     | gpg --dearmor > /etc/apt/keyrings/windsurf-stable.gpg \
  && echo "deb [arch=amd64 signed-by=/etc/apt/keyrings/windsurf-stable.gpg] https://windsurf-stable.codeiumdata.com/wVxQEIWkwPUEAGf3/apt stable main" \
     > /etc/apt/sources.list.d/windsurf.list \
  && apt-get update \
  && apt-get install -y --no-install-recommends windsurf \
  && rm -rf /var/lib/apt/lists/*

# ── Antigravity (.deb) ────────────────────────
ARG ANTIGRAVITY_DEB=https://antigravity.google/releases/antigravity_latest_amd64.deb
RUN curl -fsSL $ANTIGRAVITY_DEB -o /tmp/antigravity.deb \
  && dpkg -i /tmp/antigravity.deb || apt-get install -fy \
  && rm /tmp/antigravity.deb

# ── CUDA Toolkit (runtime のみ) ───────────────
RUN apt-get update && apt-get install -y --no-install-recommends \
  cuda-cudart-12-4 libcublas-12-4 \
  && rm -rf /var/lib/apt/lists/*

# ── user ──────────────────────────────────────
RUN useradd -m -s /bin/bash agent
USER agent
WORKDIR /home/agent/work
ENV PATH="/home/agent/.local/bin:$PATH"

COPY --chown=agent:agent entrypoint.sh /home/agent/entrypoint.sh
RUN chmod +x /home/agent/entrypoint.sh

ENTRYPOINT ["/home/agent/entrypoint.sh"]
CMD ["bash"]

🚀 entrypoint.sh

ℹ
ここはLinux版と全く同じです🐧 WSLgちゃんが画面の宛先（DISPLAY）を自動で作ってくれるので、起動する時にそれを渡すだけで動いてくれますよ✨
bash
#!/bin/bash
set -e

eval $(dbus-launch --sh-syntax)
export DBUS_SESSION_BUS_ADDRESS

openbox &

[ -e /dev/dri/renderD128 ] && export LIBGL_ALWAYS_SOFTWARE=0 \
                            || export LIBGL_ALWAYS_SOFTWARE=1
sleep 0.5

exec "${@:-bash}"

07

🔨 組み立てて動かす！ (Build & Run)

💡
compose.yml を使うとワンコマンドで起動できます: podman-compose up -d
compose.yml はリポジトリのルート（Windows向けコメントあり）に用意してあります。以下は手動で実行する場合の手順です。

イメージビルド

bash — WSL2
podman build -t agent-sandbox:latest .

コンテナ起動

bash — WSL2
podman run -it --rm \
  # WSLg の DISPLAY をそのまま渡す (Xephyr 不要)
  -e DISPLAY=$DISPLAY \
  -e WAYLAND_DISPLAY=$WAYLAND_DISPLAY \
  -v /tmp/.X11-unix:/tmp/.X11-unix:ro \
  -v $XDG_RUNTIME_DIR/$WAYLAND_DISPLAY:/tmp/$WAYLAND_DISPLAY:ro \
  \
  -v /dev/shm:/dev/shm \
  \
  # NVIDIA GPU (CDI)
  --device nvidia.com/gpu=all \
  \
  --network slirp4netns \
  \
  -v agent-workspace:/home/agent/work \
  -v ollama-models:/root/.ollama/models \
  \
  agent-sandbox:latest \
  windsurf
⚠
画面のガードについて: WSLgの画面機能をそのまま渡しているので、Linux版のXephyrちゃんほど強力なガードではありません🛡️ AIが動いている間は、Windows側で大事なパスワードを入れたりするのは控えてくださいね💦

🔍 トラブルシューティング

WSL2 内で nvidia-smi が見つからない / CUDA が使えない

Windows 側の NVIDIA ドライバーが古いか未インストールです。WSL2 は Windows 側のドライバーをそのまま使うため、WSL2 内に別途 CUDA ドライバーをインストールする必要はありません。

PowerShell
# Windows 側でドライバーバージョン確認（525.x 以上が必要）
nvidia-smi
bash — WSL2
# WSL2 内で GPU デバイスが見えるか確認
ls /dev/nvidia*
nvidia-smi
GUI ウィンドウが表示されない（WSLg が効かない）

WSLg は Windows 11 の WSL2 に自動統合されています。DISPLAY 環境変数を確認します。

bash — WSL2
echo $DISPLAY    # /tmp/... などが表示されれば WSLg 有効
xclock           # テスト — ウィンドウが開けば OK

表示されない場合は PowerShell で wsl --update を実行して WSL2 を最新版にしてください。

Podman が起動しない / "user namespace" エラー

WSL2 カーネルが古い場合に発生します。

bash — WSL2
# カーネルバージョン確認（5.15+ 推奨）
uname -r
# rootless Podman のマイグレーション
podman system migrate

解消しない場合は PowerShell で wsl --update 後に WSL2 を再起動してください。

コンテナ内で GPU が認識されない

NVIDIA Container Toolkit の CDI 定義が未生成か古くなっています。

bash — WSL2
# CDI ファイル確認
ls /etc/cdi/nvidia.yaml
# 再生成（ドライバー更新後も再実行が必要）
sudo nvidia-ctk cdi generate --output=/etc/cdi/nvidia.yaml
# GPU 認識テスト
podman run --rm --device nvidia.com/gpu=all debian:12-slim nvidia-smi
ollama pull が途中で止まる

Ctrl-C で中断して再実行すると続きからダウンロードされます。

bash — WSL2
ollama pull qwen2.5-coder:7b  # 再実行で続きから再開
