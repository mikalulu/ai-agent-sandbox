アーキテクチャ

ℹ
WSLg について: Windows 11 標準搭載の Wayland compositor + X11 サーバー。WSL2 内の GUI アプリが自動的に Windows デスクトップに表示される。Xephyr のような追加設定は不要だが、Display 隔離は Linux 版より弱い。
01

WSL2 インストール

⚠
前提: Windows 11 (Build 22000 以上)。Windows 10 の場合は WSLg が使えないため Option C (Xephyr) を推奨。

PowerShell (管理者) で実行

powershell — 管理者
# WSL2 + Ubuntu 24.04 をインストール
wsl --install -d Ubuntu-24.04

# インストール後は再起動が必要
Restart-Computer

再起動後 — Ubuntu 初期設定

powershell
# WSL2 がデフォルトになっていることを確認
wsl --set-default-version 2
wsl --list --verbose   # VERSION が 2 であることを確認
bash — WSL2 Ubuntu 内
# Ubuntu 初回起動でユーザー名・パスワードを設定後
sudo apt update && sudo apt upgrade -y

.wslconfig でリソース制限 (任意)

ini — %USERPROFILE%\.wslconfig
[wsl2]
memory=8GB          # WSL2 に割り当てる最大 RAM
processors=4        # 使用する CPU コア数
swap=4GB
gpuMemoryFraction=0.8   # GPU メモリ割り当て比率
02

CUDA / NVIDIA セットアップ

✓
WSL2 の CUDA は Windows 側ドライバーのみでよい。 Linux 用の NVIDIA ドライバーは WSL2 内にインストールしない。nvidia-container-toolkit だけ WSL2 内に入れる。

1. Windows 側 NVIDIA ドライバー確認

powershell
# Windows 側で CUDA 対応ドライバーが入っているか確認
nvidia-smi

# 未インストールの場合は winget で
winget install NVIDIA.CUDA

2. WSL2 内で nvidia-container-toolkit をインストール

bash — WSL2
# NVIDIA Container Toolkit リポジトリ追加
curl -fsSL https://nvidia.github.io/libnvidia-container/gpgkey \
  | sudo gpg --dearmor -o /usr/share/keyrings/nvidia-container-toolkit-keyring.gpg

curl -s -L https://nvidia.github.io/libnvidia-container/stable/deb/nvidia-container-toolkit.list \
  | sed 's#deb https://#deb [signed-by=/usr/share/keyrings/nvidia-container-toolkit-keyring.gpg] https://#g' \
  | sudo tee /etc/apt/sources.list.d/nvidia-container-toolkit.list

sudo apt update && sudo apt install -y nvidia-container-toolkit

# CDI デバイス定義生成
sudo nvidia-ctk cdi generate --output=/etc/cdi/nvidia.yaml

# WSL2 内から GPU が見えることを確認
nvidia-smi
03

Podman インストール

bash — WSL2
# Ubuntu 24.04 は podman が公式リポジトリにある
sudo apt install -y podman

# バージョン確認
podman --version

# rootless 動作確認
podman info | grep -i rootless
ℹ
WSL2 では systemd が使えるようになっている (Ubuntu 22.04+ / WSL2 0.67.6+)。systemctl --user enable podman.socket で rootless socket も有効にできる。
04

Dockerfile

📦
Linux 版からの差分は DISPLAY の扱いのみ。Xephyr が不要な分シンプルになる。CUDA Toolkit のバージョンは nvidia-smi で確認した CUDA Version に合わせること。
dockerfile
FROM debian:12-slim

# ── base deps ─────────────────────────────────
RUN apt-get update && apt-get install -y --no-install-recommends \
  openbox dbus-x11 xdg-utils xauth \
  libgbm1 libgl1-mesa-dri \
  libnss3 libatk1.0-0 libatk-bridge2.0-0 \
  libgdk-pixbuf2.0-0 libgtk-3-0 \
  libasound2 libxss1 libxtst6 \
  fonts-noto-cjk fonts-noto-color-emoji \
  ca-certificates curl wget gnupg \
  build-essential git \
  python3 python3-pip python3-venv \
  && rm -rf /var/lib/apt/lists/*

# ── Node.js (nvm) ─────────────────────────────
ENV NVM_DIR=/opt/nvm NODE_VERSION=22
RUN mkdir -p $NVM_DIR \
  && curl -fsSL https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.1/install.sh | bash \
  && . $NVM_DIR/nvm.sh \
  && nvm install $NODE_VERSION && nvm alias default $NODE_VERSION
ENV PATH="$NVM_DIR/versions/node/v${NODE_VERSION}/bin:$PATH"

# ── Rust ──────────────────────────────────────
ENV RUSTUP_HOME=/opt/rustup CARGO_HOME=/opt/cargo
RUN curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs \
  | sh -s -- -y --no-modify-path
ENV PATH="$CARGO_HOME/bin:$PATH"

# ── CLI AI agents ─────────────────────────────
RUN npm install -g \
  @anthropic-ai/claude-code \
  @openai/codex \
  @google/gemini-cli \
  opencode-ai

RUN pip3 install --break-system-packages aider-chat

# ── Ollama (binary only) ──────────────────────
RUN curl -fsSL https://ollama.com/install.sh | sh

# ── Windsurf (.deb) ───────────────────────────
ARG WINDSURF_DEB=https://windsurf-stable.codeiumdata.com/linux-x86_64/windsurf_latest_amd64.deb
RUN curl -fsSL $WINDSURF_DEB -o /tmp/windsurf.deb \
  && dpkg -i /tmp/windsurf.deb || apt-get install -fy \
  && rm /tmp/windsurf.deb

# ── Antigravity (.deb) — URL 公開後に差し替え ─
ARG ANTIGRAVITY_DEB=https://antigravity.google/download/antigravity_latest_amd64.deb
RUN curl -fsSL $ANTIGRAVITY_DEB -o /tmp/antigravity.deb \
  && dpkg -i /tmp/antigravity.deb || apt-get install -fy \
  && rm /tmp/antigravity.deb

# ── CUDA Toolkit (runtime のみ) ───────────────
RUN apt-get update && apt-get install -y --no-install-recommends \
  cuda-cudart-12-4 libcublas-12-4 \
  && rm -rf /var/lib/apt/lists/*

# ── user ──────────────────────────────────────
RUN useradd -m -s /bin/bash agent
USER agent
WORKDIR /home/agent/work

COPY --chown=agent:agent entrypoint.sh /home/agent/entrypoint.sh
RUN chmod +x /home/agent/entrypoint.sh

ENTRYPOINT ["/home/agent/entrypoint.sh"]
CMD ["bash"]
05

entrypoint.sh

ℹ
Linux 版との差分なし。WSLg が DISPLAY を自動設定するため、コンテナ起動時に -e DISPLAY=$DISPLAY を渡すだけで動作する。
bash
#!/bin/bash
set -e

eval $(dbus-launch --sh-syntax)
export DBUS_SESSION_BUS_ADDRESS

openbox &

[ -e /dev/dri/renderD128 ] && export LIBGL_ALWAYS_SOFTWARE=0 \
                            || export LIBGL_ALWAYS_SOFTWARE=1
sleep 0.5

exec "${@:-bash}"

Build & Run

イメージビルド

bash — WSL2
podman build -t agent-sandbox:latest .

コンテナ起動

bash — WSL2
podman run -it --rm \
  # WSLg の DISPLAY をそのまま渡す (Xephyr 不要)
  -e DISPLAY=$DISPLAY \
  -e WAYLAND_DISPLAY=$WAYLAND_DISPLAY \
  -v /tmp/.X11-unix:/tmp/.X11-unix:ro \
  -v $XDG_RUNTIME_DIR/$WAYLAND_DISPLAY:/tmp/$WAYLAND_DISPLAY:ro \
  \
  -v /dev/shm:/dev/shm \
  \
  # NVIDIA GPU (CDI)
  --device nvidia.com/gpu=all \
  \
  --network slirp4netns \
  \
  -v agent-workspace:/home/agent/work \
  -v ollama-models:/root/.ollama/models \
  \
  agent-sandbox:latest \
  windsurf
⚠
Display 隔離について: WSLg の DISPLAY を直接渡しているため、Linux 版の Xephyr ほど隔離は強くない。コンテナ起動中はホスト側での機密操作を避ける運用を推奨。
08

Ollama

✓
Linux 版と同様にコンテナ内で動作。WSL2 では CUDA が透過的に使えるため、GPU 推論が有効になる。

コンテナ内で起動 · モデル取得

bash — コンテナ内
ollama serve &
sleep 2

# コーディング特化モデル
ollama pull qwen2.5-coder:7b   # ~5GB · GPU 推論

# GPU 使用確認
ollama ps   # SIZE (VRAM) が表示されれば GPU 動作中

VRAM 使用確認 (WSL2 側)

bash — WSL2
nvidia-smi --query-gpu=memory.used,memory.free --format=csv

Tips

WSL2 の再起動

powershell
wsl --shutdown          # 全 distro 停止
wsl -d Ubuntu-24.04     # 再起動

Windows Terminal から直接コンテナ起動するプロファイル

json — WT settings.json の profiles.list に追加
{
  "name": "Agent Sandbox",
  "commandline": "wsl.exe -d Ubuntu-24.04 -- bash -c 'podman start -ai agent-sandbox'",
  "icon": "🤖",
  "startingDirectory": "//wsl.localhost/Ubuntu-24.04/home/YOUR_USER"
}

ワークスペースのバックアップ

bash — WSL2
podman run --rm \
  -v agent-workspace:/work:ro \
  -v $(pwd):/out \
  debian:12-slim \
  tar czf /out/workspace-$(date +%Y%m%d).tar.gz -C /work .

CUDA バージョン整合性確認

bash — WSL2
# Windows ドライバーの CUDA Version を確認して Dockerfile を調整
nvidia-smi | grep "CUDA Version"
