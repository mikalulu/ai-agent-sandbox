🦭 Podman が推奨な理由

デーモンレスで常に rootless 動作するため、dockerd の root 権限による socket hijack リスクがありません。slirp4netns による NAT は外部への通信を許可しつつ、コンテナからホスト内ネットワークへのアクセスを遮断します。NVIDIA CDI も公式サポートされており、GPU AI Sandbox にとって最も設定が整っています。

🐳 Docker の注意点

dockerd は root で動作するため、/var/run/docker.sock にアクセスできると実質 root 権限が取れます。コンテナ内から docker.sock をマウントすると「コンテナ脱出」が容易になるため、AI Sandbox 環境ではマウントしないことが必須です⚠

📦 LXC を使う場面

フル OS 環境 (systemd 込み) が必要なテスト、あるいは Proxmox 上の仮想化として使われます。AI エージェントの軽量サンドボックスには重すぎますが、開発環境まるごとを隔離したい場合には Lima (macOS) と並ぶ選択肢になります🐧

🔐 Secure Element (SE) vs HSM

SE はスマートカードや SIM のチップです。HSM より小型で安価ですが、処理速度は低めです。どちらも鍵が外部に漏洩しない点は共通です。

🛡️ SELinux vs AppArmor

SELinux はラベルベースで細粒度の制御を行います。AppArmor はパスベースで、設定が読みやすく自動生成にも対応しています。

🧅 多層防御 (Defense in Depth)

実環境では単一の手法に頼らず、複数の防御層を重ねます。手法を組み合わせることで攻撃面を段階的に縮小できます。

👤 ユーザー制御層の注意点

ユーザー制御層の機能は管理者権限で変更可能です。root 権限を奪取されると無効化されるリスクがあるため、MAC 層など他の隔離手法との併用が必要です。
